mag.import_graph()

从文件导入一个图到系统中，可以是ZIP包或JSON配置文件。

入参

  • file_path - 图文件的路径，可以是ZIP包或JSON文件

预期输出

返回值

导入的图的名称或ID

使用示例

import mag

# 从ZIP包导入图
graph = mag.import_graph("easy_search.zip")
print(f"已导入图: {graph}")
mag.list_graphs()

获取当前系统中所有图的列表。

入参

无参数

预期输出

返回值

包含图名称或ID的列表

使用示例

import mag

# 获取所有图
graphs = mag.list_graphs()
print(f"找到 {len(graphs)} 个图: {graphs}")
mag.get_graph()

查看指定图的详细信息和配置。

入参

  • graph_name - 要查看的图的名称或ID

预期输出

返回值

包含图配置信息的字典或对象

使用示例

import mag

# 查看指定图的配置
result = mag.get_graph("news_processing_workflow")
print("图配置信息:")
print(result)
mag.save_graph()

保存或更新图配置。如果图不存在则创建新图，如果已存在则更新配置。

入参

  • config - 图的配置字典，必须包含name字段

预期输出

返回值

操作结果字典

使用示例

import mag

# 定义一个简单的图配置
graph_config = {
    "name": "simple_chat",
    "description": "简单的聊天图",
    "nodes": [
        {
            "id": "chat_node",
            "name": "聊天节点",
            "type": "agent",
            "model_name": "gpt-4",
            "system_prompt": "你是一个友好的助手",
            "user_prompt": "请回复用户的问题",
            "mcp_servers": [],
            "position": {"x": 100, "y": 100}
        }
    ],
    "edges": [],
    "version": "1.0"
}

# 保存图
result = mag.save_graph(graph_config)
print(f"保存结果: {result}")
mag.delete_graph()

从系统中删除指定的图。

入参

  • graph_name - 要删除的图的名称或ID

预期输出

返回值

删除操作的结果或确认消息

使用示例

import mag

# 删除指定的图
result = mag.delete_graph("news_processing_workflow")
print("删除结果:", result)
mag.run_graph()

运行指定的图并获取执行结果。

入参

  • graph_name - 要运行的图的名称或ID
  • input_text - 图运行所需的输入文本
  • parallel - 是否并行执行，默认为False（可选）

预期输出

返回值

图运行的结果或处理后的数据

使用示例

import mag

# 运行图并获取结果
result = mag.run_graph("news_processing_workflow", input_text="处理这个任务")
print("图运行结果:", result)
mag.continue_run()

继续执行现有的会话，可以从断点恢复或添加新的输入继续处理。

入参

  • conversation_id - 会话ID
  • input_text - 新的输入文本（可选）
  • parallel - 是否并行执行，默认为False（可选）
  • continue_from_checkpoint - 是否从断点继续，默认为False（可选）

预期输出

返回值

执行结果字典

使用示例

import mag

# 首先运行一个图
initial_result = mag.run_graph("interactive_workflow", "开始处理数据")
conversation_id = initial_result['conversation_id']

# 继续会话，添加新输入
continue_result = mag.continue_run(conversation_id=conversation_id, input_text="现在进行深度分析")
print(f"继续执行结果: {continue_result}")
mag.export_graph()

将图及其依赖项导出为ZIP包，包含配置、提示词、文档等所有相关文件。

入参

  • graph_name - 要导出的图的名称或ID

预期输出

返回值

导出结果，包含导出文件路径

使用示例

import mag

# 导出图
result = mag.export_graph("research_workflow")
if result['status'] == 'success':
    print(f"✅ 导出成功! 文件路径: {result['file_path']}")
else:
    print(f"❌ 导出失败: {result.get('message', 'Unknown error')}")
mag.generate_mcp_script()

为指定图生成MCP脚本，支持并行和顺序执行两种模式。

入参

  • graph_name - 要生成脚本的图的名称或ID

预期输出

返回值

包含并行和顺序执行脚本内容的字典

使用示例

import mag

# 生成MCP脚本
result = mag.generate_mcp_script("research_workflow")

# 保存并行执行脚本
with open("research_workflow_parallel.py", "w", encoding="utf-8") as f:
    f.write(result["parallel_script"])

# 保存顺序执行脚本
with open("research_workflow_sequential.py", "w", encoding="utf-8") as f:
    f.write(result["sequential_script"])

print("✅ MCP脚本生成完成")
mag.get_prompt_template()

生成包含当前系统中所有可用工具和模型信息的提示词模板，用于辅助LLM生成图配置。

入参

无参数

预期输出

返回值

包含工具和模型信息的提示词模板字符串

使用示例

import mag

# 获取提示词模板
template = mag.get_prompt_template()

# 保存模板到文件
with open("mag_prompt_template.md", "w", encoding="utf-8") as f:
    f.write(template)

print("✅ 提示词模板已生成: mag_prompt_template.md")
